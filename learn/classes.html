Classes


Now we begin OOP in Zuko. To be able to create objects you need to define the blueprint of each object. For that purpose classes are used. A class can have variables or functions as it’s members. These members can be private or public. Members not specified to be either public or private are public by default. Private members can only be accessed from within the class whereas public members can also be accessed from outside the class. The member operator ‘.’ is used to access the members of an object.

Example
class rect
{
    private var length = nil
    private var width = nil
    var area = nil # public by default
    function __construct__(var l,var w)
    {
      self.length = l
      self.width = w
      self.area = self.length * self.width
    }
    function getArea()
    {
      return self.area
    }
}
var r = rect(10,20)
println(r.area)
println(r.getArea())


Output
200
200


  • The constructor of a class is named as __construct__
  • self is object from which the function was called
When you call obj.foo() Zuko calls foo() from obj but also adds the object itself as an argument i.e obj.foo(obj) . This object is accessed inside member functions by name self

Here is another example:

# Simple Vector Object
# Implemented using very basic concepts
# of OOP
class Vector
{
    private var a = nil
    private var b = nil
    private var c = nil
    function __construct__(var a,var b,var c)
    {
    self.a = a
    self.b = b
    self.c = c
    }
    function add(var a)
    {
    self.a += a.a
    self.b += a.b
    self.c += a.c
    }
    function display()
    {
    println(self.a,"i + ",self.b,"j + ",self.c,"k")
    }
}
var v1 = Vector(2,3,5)
var v2 = Vector(3,2,0)
v1.add(v2)
v1.display()


Output

5i+5j+5k
