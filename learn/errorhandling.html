Error handling


Plutonium is a managed language which means that there is no undefined behavior and errors are reported as soon as they occur along with their line number. However sometimes you know that an error may occur and you want to execute some other code in case an error occurs instead of the interpreter just showing the error and exiting. Basically you donâ€™t want your program to blow up and want it to do something to tackle an error. Error handling is supported by Plutonium. You can use try catch statement to handle errors. However, remember that we are talking about runtime errors. You cannot catch Syntax Errors in your code. The code that might throw errors is placed inside the try block and catch block has the code that you want to run in case an error occurs.

There are some global classes already defined by plutonium. These are called error classes. There is a base class named Error and all other error classes are derived from it. When an error is thrown it is a specific error. For example,TypeError is different from MathError. The base class Error has only one public member msg. As all other error classes are derived from it so they also have this member.

Example
try
{
    print("Enter a: ")
    var a = int(input())
    print("Enter b: ")
    var b = int(input())
    println(a/b)
}
catch(err)
{
    println("caught error!")
    # err is nothing but an object of any one of the error classes
    # err will always have a member msg
    println(err.msg)
    println(err) # should print MathError object

}
println("all good")


Try entering 0 when asked to input b You would get the following output

Enter a: 10
Enter b: 0
caught error!
Error division by zero
<MathError Object>


In the above example MathError was thrown by division operator. Plutonium code can throw other kinds of errors too. For example, an ImportError is thrown when there is an issue importing a module. Every runtime error has a class of it's own.

Error classes are global variables defined with same names as above. You can also manually throw an error of any type using throw statement.

Consider the following example

try
{
    var i = input("Enter an integer or float: ")
    if(!isnumeric(i)) # string can not be converted to a float or integer
      throw ValueError("Invalid Input!")
    i = tonumeric(i)
    # proceeed
}
catch(err)
{
    println(err)
    println(err.msg)
}


Output
Enter an integer or float: no
<ValueError Object>
Invalid Input!


In the above example a ValueError was thrown by throw statement. The throw statement takes an object of any error class.
