The Standard Library


Zuko interpreter provides a very minimal list of built-in functions. Because these built- in functions are always loaded when the interpreter starts .Loading a lot of functions most of which will never be used is a bad idea so modules are introduced . A module is also some code (functions and classes) which you can import and use. Modules are not imported by default. You have to tell zuko which module to load. By using this module system you only load what you are going to use.

There are two types of modules in Zuko.

Zuko Code files:


These are simple zuko files having extension “.plt”. In topic Multifile Programs you were taught how to import other zuko files into your program and use them. The same concepts apply here. There you created a new zuko file of your own and imported it into current program. There are a few zuko files in C:\plutonium\std These come along with the interpreter and can be imported and used as well. These files contain functions for common algorithms. Currently the std folder has files algo.plt, json.plt and ll.plt To import any of these files you can use statement

import "C:\\plutonium\\std\\algo.plt" # windows
import "/opt/plutonium/std/algo.plt" #linux
        
# or you can use following syntactic sugar
        
import std/algo.plt # works on both windows and linux

All functions of each of these files are inside a namespace having the same name as the file. For example,to use bubbleSort function from algo.plt you would use name algo::bubbleSort()

Following is some documentation about these libraries

algo.plt

  • bubbleSort(arr) → sorts the array
  • selectionSort(arr) → sorts the array
  • insertionSort(arr) → sorts the array
  • binarySearch(arr,x) → searches the sorted array for element x
  • linearSearch(arr,x) → searches the array for element x
  • quickSort(arr,l,h) → sorts the array from index l to index h
  • mergeSort(arr,l,h) → sorts the array from index l to index h
  • subList(l,h,list) → returns sublist of list starting from index l to index h
  • filter(fn,A) → returns a list of all those elements of list A that make fn function return true
  • apply(fn,list) → applies fn function to all elements of the list
  • Max(list) → returns the max from list
  • Min(list) → returns the min from list•
  • count(fn,list) → returns the number of elements in list that make fn function true
  • first_mismatch(list1,list2) → returns the index where first mismatch occurs
  • transpose(matrix) → returns the transpose of matrix represented as 2d list
  • count_set_bits(x) → returns number of bits set in x
  • isPowerOfTwo(x) → returns true is x is power of 2
  • median(list) → returns the median
  • hex(n) → returns a string representing hex value of n
  • octal(n) → returns a string representing octal value of n
  • bin(n) → returns a string representing binary value of n
  • encodeASCII(str) → encodes a string as ascii and returns a bytearray

ll.plt


This file implements a doubly linked list. It has two classes Iterator and List. The List class creates instances of Iterator class. You won't have to create objects of Iterator class ever.

The Iterator class has following methods

  • next() → moves the iterator to next node
  • prev() → moves the iterator to previous node
  • value() → returns the value of current node iterator is pointing to
  • __eq__(rhs) → equality operator overloaded
  • __noteq__(rhs) → inequality operator overloaded

The List class has following methods
  • __construct__() → constructor takes 0 arguments
  • insert(it,val) → inserts the val at node pointed by iterator and the previous node is pushed to the right
  • setvalue(it,val) → sets the value of node pointed by iterator
  • erase(it) → erases the node pointed by iterator
  • push_back(val) → adds value at the end of the list
  • push_front(val) → adds value the start of the list
  • begin() → returns an iterator pointing to first node
  • end() → returns an iterator pointing to dummy tail node

Example

import std/algo.plt
import std/ll.plt
var arr = [5,4,3,2,1]
algo::selectionSort(arr)
println(arr)
var nums = ll::List()
nums.insert(nums.end(),1)
nums.insert(nums.end(),2)
nums.insert(nums.end(),3)
var it = nums.begin()
while(it!=nums.end())
{
    println(it.value())
    it.next()
}

Native Modules:


These are modules written in C++ and not in Zuko. These modules are present in C:\ plutonium\modules\ or /opt/plutonium/modules depending on the OS. These shared objects(dlls in case of windows) .To import any of these you just simply use the following syntax:

import modulename

Note that no “.plt” or double quotes are added here. After loading a native module you use the member operator '.' to access it's members like you do with objects. For example, to call sin function from math module you write math.sin(@pi/2). Currently zuko has following native modules:

math


The math module provides functions related to maths. It has no classes. Following are it’s members:

  • sin(x) → returns sin of x
  • cos(x) → returns cos of x
  • tan(x) → returns tan of x
  • asin(x) → returns sin inverse of x
  • acos(x) → returns cos inverse of x
  • atan(x) → returns tan inverse of x
  • sinh(x) → returns sin hyperbolic of x
  • cosh(x) → returns cos hyperbolic of x
  • tanh(x) → returns tan hyperbolic of x
  • asinh(x) → returns hyperbolic sin inverse of x
  • acosh(x) → returns hyperbolic cos inverse of x
  • atanh(x) → returns hyperbolic tan inverse of x
  • log(x) → returns natural log of x
  • log10(x) → returns base 10 log of x
  • trunc(x) → truncates and returns float x
  • round(x) → rounds and returns float x
  • ceil(x) → returns ceil of float x
  • floor(x) → returns floor of float x
  • radians(x) → converts x from degrees to radians
  • sqrt(x) → returns sqrt of x

regex


regex module provides regular expression support in zuko. It has following 3 functions:

  • search(str,patt) → searches the parts of string that match regex and returns a list of these parts
  • match(str,patt) → returns true if string matches regex pattern
  • replace(str,patt) → replaces parts of string that match regex pattern

conio


conio module provides old conio.h functions .It has following 2 functions:

  • kbhit() → returns true is key is pressed
  • getch() → gets a key from user and returns it's scan code as integer

console


console module provides some winapi functions to work with the console. It has following 4 functions:

  • hide() → hides console windows
  • show() → shows console window
  • setTextAttribute(k) → sets text attribute
  • gotoxy(x,y) → moves cursor to xth row and yth column
  • getDimensions() → returns a list containing number of rows and columns in console

bignum


bignum module provides arbitrary precision integers support in zuko. It has only one class called bignum. Following are the members of this class:

  • __construct__(x) → constructor takes a string argument containing digits only
  • __add__(rhs) → addition operator overloaded
  • __sub__(rhs)
  • __mul__(rhs)
  • __div__(rhs)
  • __smallerthan__(rhs)
  • __greaterthan__(rhs)
  • __smallerthaneq__(rhs)
  • __greaterthaneq__(rhs)
  • __eq__(rhs)
  • __noteq__(rhs)

turtle


turtle module provides turtle graphics support in zuko. It has following functions:

  • setWindowSize(rows,cols) → sets the window size
  • setColor(r,g,b) → sets the turtle color
  • setBgColor(r,g,b) → sets window’s bg color
  • forward(k) → moves turtle k units forward
  • back(k) → moves k units back
  • left(k) → turns k degrees left
  • right(k) → turns k degrees right
  • circle(r) → draws a circle of radius r with current position as center
  • penUp() → lifts up the pen
  • penDown() → ...
  • done() → sets the window in idle state
  • setPenSize() → sets the pensize
  • goto(x,y) → goes to xth row and yth column without drawing anything
  • moveto(x,y) → goes to x,y and draws the path
  • show() → shows turtle
  • hide() → hides turtle
  • setSpeed(x) → sets the turtle speed(pixels per frame)

autogui


autogui module allows you to perform GUI tasks with code. It has following functions:

  • getMousePos() → returns current screen coordinates of mouse
  • mouse_leftClick(x,y) → moves to x,y and left clicks mouse
  • mouse_rightClick(x,y) → ..
  • getKeyCodes() → returns a dictionary representing the keycode of function keys
  • typewrite(str) → typewrites alphanumeric string

mysql


mysql module allows you to connect to a mariadb/mysql server. It has following functions:

  • init() → returns a connection object
  • real_connect(conn,host,user,pass,db,port = 3306) → connects a connection object to a mysql server. If the port is unspecified then it is 3306 by default.
  • query(conn,query) → Executes a query string using the established connection
  • store_result(conn) → returns a result object representing the result of last query executed using connection conn
  • nums_row(res) → returns the number of rows in the result object
  • num_fields(res) → returns the number of fields in the result object
  • fetch_row_as_str(res) → fetches a row from res and returns a list of string values
  • fetch_row(res) → fetches a row from res and returns a list of bytearrays
  • close(conn) → closes the connection

cgi


cgi module allows you to do CGI web programming in zuko. Currently it supports GET and POST(urlencoded + multipart) methods. It has following functions:

  • FormData() → returns a dictionary representing the form data(html form elements’ names as keys and their values as dictionary values).

libcurl


Libcurl is a networking library originally written in C. This module is libcurl’s binding for zuko. It has following classes:

Curl
  • __construct__() → constructor
  • setopt(opt,val) → sets value of an option
  • perform() → performs the network task and returns an integer CURLCode
  • getinfo(x) → Returns info
  • __del__() → object deletor
mime
  • __construct__(curlObj) → creates a mime object for a curl object and returns it
  • addpart() → adds a part to mime and returns a mime part object
mimepart
  • __construct__() → constructor
  • name(str) → sets the name of mimepart
  • data(bytes) → sets data of the mimepart. Bytes is a bytearray
  • filename(str) → sets filename of the mimepart.

Other members of this module are:

  • OPT_URL → some constant value
  • OPT_FOLLOWLOCATION → ..
  • OPT_VERBOSE → ..
  • OPT_POSTFIELDS → ..
  • OPT_MIMEPOST → ..
  • OPT_PORT → ..
  • OPT_WRITEFUNCTION → ..
  • INFO_HTTP_CODE → ..
  • INFO_CONTENT_TYPE → ..
  • CURLE_OK → value returned by Curl.perform() on success
  • VERSION_MAJOR → Integer major version
  • VERSION_MINOR → integer minor version
  • WriteMemory → constant value
  • strerror(x) → returns a string message corresponding to error code x, where x is returned by Curl.perform()


The constant values are used to specify which option to set or which info to get when using setopt() and getinfo() functions.

Example 
import libcurl as lc
import std/algo.plt
var curl = lc.Curl()
curl.setopt(lc.OPT_URL,"https://www.google.com")
curl.setopt(lc.OPT_FOLLOWLOCATION,1)
curl.setopt(lc.OPT_WRITEFUNCTION,lc.WriteMemory)
#use default WriteMemory function
#it comes with the module
var res = curl.perform()
if(res != lc.CURLE_OK) #error occurred
{
    println(lc.strerror(res))
    exit()
}
#request was successful
#curl.data stores response data as bytearray
#decode it as ascii bytes, only do this when response is text
#even if the text is utf8 it will be atleast printed correctly
var str = algo::decodeASCII(curl.data)
println(str)

The above example fetches the url “https://www.google.com” .The assumption here is that google will return a text/html response. You can check the HTTP code and content type of response using getinfo() if you want to go more advanced. Also, there is no cleanup method in curl object .The cleanup is done automatically by the module when the object deletor __del__() is called.
